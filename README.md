# Система оптимизации распределения электрической энергии

Проект решает задачу потокораспределения в электрической сети "Альфа" компании Транснефть. Находит оптимальный план доставки энергии от источников к потребителям с учётом ограничений пропускной способности линий электропередачи.

## Как это работает

Проект реализует двухэтапный подход:

### Этап 1: ML-приближение
Нейросеть предсказывает начальное распределение потоков по путям за миллисекунды. Обучается на синтетических данных, сгенерированных методом Latin Hypercube Sampling.

### Этап 2: Точный расчёт (солвер)
Градиентный спуск уточняет решение, начиная с ML-приближения или равномерного распределения. Использует двухэтапное обновление и пропорциональное урезание при дефиците пропускной способности.

### Ключевые особенности
- Гибридный подход: скорость ML + точность оптимизации
- Интерактивная визуализация потоков на графе сети
- Веб-интерфейс для удобной работы (Streamlit)
- Генерация синтетических данных для обучения
- Сохранение и загрузка обученных моделей

## Режимы работы

Веб-интерфейс предоставляет три режима:

### 1. ML-обучение

Обучает нейросеть с нуля. Пользователь задает параметры:
- Количество сценариев для обучения
- Уровни разреженности (доля ребер с бесконечной capacity)
- Масштабы заявок
- Параметры обучения (эпохи, learning rate, размер батча)
- Веса функции потерь

Результат: сохраненная модель `genereted/model.pt`, графики обучения.

### 2. ML-предсказание

Использует обученную модель для быстрого предсказания потоков. Пользователь загружает файл модели (.pt). Результат: интерактивный граф с распределением потоков, таблица загрузки ребер.

### 3. Точное решение (ML + солвер)

Запускает градиентный спуск, используя ML-предсказание как начальное приближение. Пользователь задает параметры солвера (learning rate, количество итераций, epsilon сходимости). Результат: уточненное оптимальное решение с визуализацией.

## Входные данные

### edges.json

Список ребер графа с пропускной способностью:

```json
[
  {"nodes": ["P", "XI."], "capacity": 2670.3},
  {"nodes": ["37", "VIII."], "capacity": "inf"}
]
```

- `"inf"` означает неограниченную пропускную способность

### flows.json

Заявки от источников к потребителям:

```json
{
  "A": {
    "1": 738.0,
    "3": 1200.0,
    "4": 420.0
  }
}
```

- Ключи верхнего уровня — источники (латинские буквы)
- Ключи второго уровня — потребители (цифры)
- Значения — требуемая мощность в кВт

## Интерактивный граф

При наведении на элементы графа отображается подробная информация:

**Узел-источник:** список потребителей с объемами доставки (заявлено / фактически)

**Узел-потребитель:** список источников с объемами получения

**Ребро:** направление потока, детальные значения в обе стороны, суммарный поток, загрузка в процентах

Цвет ребра зависит от загрузки:
- Зеленый: менее 60%
- Желтый: 60-80%
- Оранжевый: 80-95%
- Красный: 95-100%
- Бордовый: более 100% (перегрузка)

## Конфигурация

### settings/run_config.json

| Поле | Значения | Описание |
|------|----------|----------|
| `mode` | `"train"`, `"predict"`, `"solve"` | Режим работы |
| `use_ml_initial_guess` | `true`/`false` | Использовать ML для начального приближения |
| `model_path` | путь к файлу | Путь к сохраненной модели |
| `visualize_flows` | `true`/`false` | Создавать HTML-граф |

### settings/config.json

| Секция | Основные параметры |
|--------|-------------------|
| `training` | num_samples_per_level, sparsity_levels, demand_scale_factors, batch_size, epochs, learning_rate |
| `model` | hidden_dims, dropout_rate |
| `loss` | demand_weight, excess_weight, capacity_weight |
| `solver` | learning_rate, max_iter, epsilon, capacity_weight |
| `paths` | generated_folder, model_save_name |
